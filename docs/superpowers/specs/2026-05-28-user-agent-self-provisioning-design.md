# 用户自助创建私有智能体设计

**日期:** 2026-05-28  
**状态:** 已确认（实施中）  
**目标:** 用户登录后不再自动初始化全部智能体，改为在 `AgentSelector` 中由用户基于模板自助创建私有智能体

---

## 1. 需求边界（已确认）

- 登录后不自动为用户初始化全部智能体。
- 普通用户通过 `AgentSelector` 的创建入口，自助创建私有智能体。
- 创建方式为“基于模板创建私有副本”，并允许完整高级配置。
- 首次登录若无任何私有智能体，显示空状态并提示创建。
- 可选模板范围：默认所有已维护且启用的模板。
- 本次优先改造后端 A 层（用户初始化策略），前端登录后 `AgentSelector` 拉取机制保持现有方式，但其返回语义改为“当前用户私有列表”。

---

## 2. 方案选择与取舍

### 2.1 备选方案

- 方案 A：模板目录 + 用户私有实例双层模型（长期最优，改动面较大）。
- 方案 B：最小演进（推荐本次采用）：
  - 取消注册时全量 seed；
  - 增加“从模板创建私有副本”能力；
  - 保持现有主要路由与页面结构。
- 方案 C：前端伪分层（不推荐，权限和一致性风险高）。

### 2.2 采用方案

采用 **方案 B**，优先满足“用户不再自动拥有全部智能体、按需自助创建”目标，后续可平滑演进到完整双层模型。

---

## 3. 架构与权限边界

### 3.1 核心实体

- 模板智能体（管理员维护）：
  - 由管理员在智能体管理页维护；
  - 提供给普通用户作为创建来源。
- 用户私有智能体（用户持有）：
  - 用户基于模板创建；
  - 归属当前用户，供 `AgentSelector` 与会话切换使用。

### 3.2 权限策略

- 普通用户：
  - 可读取“我的私有智能体列表”；
  - 可读取“可创建模板列表（启用模板）”；
  - 可调用“从模板创建私有副本”接口；
  - 不可调用模板管理写接口（保持 `403`）。
- 管理员：
  - 继续维护模板池（新增/编辑/禁用/删除）；
  - 不受普通用户私有实例行为影响。

---

## 4. 数据流与 API 设计

### 4.1 前端数据流（`AgentSelector`）

1. 初始化调用 `GET /api/agents`，获取“当前用户私有智能体”。
2. 若列表为空：
   - 下拉面板展示空状态文案；
   - 提供 `创建智能体` 按钮。
3. 点击创建后：
   - 拉取模板列表；
   - 选择模板；
   - 进入完整配置表单（名称、描述、模型、技能、工作区等）；
   - 提交创建；
   - 刷新列表并自动选中新实例。

### 4.2 API 建议

- `GET /api/agents`
  - 普通用户：返回该用户私有智能体列表（可为空）。
  - 管理员：保持现有模板管理视角（本次实现可保持兼容）。
- `GET /api/agent-templates`（新增）
  - 返回可用于创建的模板列表（默认所有启用模板）。
- `POST /api/agents/from-template`（新增）
  - 入参：
    - `template_agent_id`（必填）
    - 用户覆盖配置（名称、描述、模型、技能、工作区等，可选）
  - 行为：
    - 以模板为基线创建用户私有副本；
    - 保存用户维度配置与工作区目录；
    - 返回新私有实例摘要。

### 4.3 存储策略

- 继续沿用每用户目录体系（`agent_configs` / `agent_data` / `agent_workspaces`）。
- 移除注册流程中的“全量 seed”调用，改为“创建时按需落盘”。
- 私有智能体 ID 建议增加前缀，如 `u_<shortid>`，避免与模板 ID 冲突。

---

## 5. 关键异常与边界场景

- 空列表首登：
  - `GET /api/agents` 返回空数组时不报错；
  - 前端显示空态并引导创建。
- 模板变更：
  - 模板被禁用/删除时，`POST /api/agents/from-template` 返回 `404/409`；
  - 前端提示“模板已变更，请重新选择”并刷新模板。
- 并发创建：
  - 防重复提交（幂等键或名称冲突策略）；
  - 避免生成重复私有实例。
- 越权保护：
  - 普通用户调用模板管理写接口一律 `403`。
- 配置兼容：
  - 模板默认模型/技能不满足当前环境时，返回可读错误，前端保留表单供修正。
- 会话安全：
  - 退出登录时清理 agent 缓存状态，避免跨账号残留。

## 6. 测试计划与验收标准

### 6.1 验收标准

1. 普通用户首次登录不再自动拥有全部智能体。
2. 普通用户在 `AgentSelector` 可见空态与创建入口。
3. 普通用户可基于启用模板创建私有副本并正常切换使用。
4. 普通用户无法操作模板管理写接口（`403`）。
5. 管理员模板管理能力不回退。

### 6.2 后端测试

- 用户注册后不触发全量 `seed_all_agents_for_user`。
- `GET /api/agents`：
  - 普通用户返回私有列表（可空）；
  - 管理员行为符合既有约束。
- `GET /api/agent-templates` 仅返回启用模板。
- `POST /api/agents/from-template`：
  - 正常创建成功；
  - 模板禁用/不存在返回 `404/409`；
  - 非法覆盖字段被拒绝或忽略（按接口定义）。

### 6.3 前端测试

- `AgentSelector` 空列表时渲染空态与创建入口。
- 创建流程完成后自动刷新并选中新实例。
- 模板失效、网络失败、校验失败均有清晰提示。
- 切换账号后不会残留上一用户 agent 状态。

### 6.4 回归测试

- Chat/Coding 在已有私有智能体时行为保持一致。
- 管理员 `/agents` 页面模板管理功能不回退。
- 多用户隔离目录结构与加载流程保持兼容。

---

## 8. 风险与缓解

- 风险：列表语义变更影响现有页面假设（默认总有 `default`）。
  - 缓解：补齐空列表分支与无选中态处理。
- 风险：模板配置与当前环境不兼容导致创建失败率高。
  - 缓解：创建表单提供完整高级配置并加强错误提示。
- 风险：并发创建造成重复私有实例。
  - 缓解：增加后端幂等控制与唯一性校验。
